Quote Online

Quote Online is a huge web application for a pool company that I have been doing for about 8 months. Javascript and jQuery have been used for front-end and PHP for back-end. Every page is responsive and looks good on all devices. Features:
  • Create employees account
  • Create/update client
  • Create/update pool/spa project
  • Messaging system
  • Contract creation
  • Presentation page

Creation of new employees account

Admin can create a new employees account by filling out the form that is shown on the photo. Every form element is validated and if the information is missing or it is invalid, error message is shown.

Employee front page

Employee front page shows messages that are forwarded to the employee or the employee's company by the admin, or a company owner.

Message System

Message system enables admin to send messages that will be shown on the front page of the employees. Message can be sent to entire companies or the selected employees page. This is temporary solution and it is planed to improve this system in the future.

Working with client information

Employee account can create/update client. If the employee picks a client to update, entire form is updated with existing information and can be changed. Clients status can be changed to active, lost sale or inactive. Every form is validated and error messages are shown if the information is missing or invalid.

Working with pool information

Employee chooses a client and project from the dropdowns and the page is populated with the existing information. New project for a client can be created. Every form field is validated and error messages are shown.

If the employee did not choose a client and a projects and tries to access other pages, modal for choosing clients and projects will appear. The employee can't access pages till the client and project have been choosen.

Challenge here was population of the pool information based on the selected project, or more precisely appearing and disappearing of specific form fields based on other choices.

On some pages there is a option to leave a note or bid. If the employee clicks the button, the add bid/note modal appears. On the left default modal is shown and on the right the filled modal.

When the pool project is finished, the employee can create a presentation. Cover photo must be uploaded as well as plan for presentation. Additional photos can be uploaded (up to 6 photos). This will be shown on the presentation page.

Presentation page shows all the choosen items of the created pool/spa project with descriptions and prices for each one. Total price is calculated that includes tax and other charges.

The challenge that occured on the presentation page was how to make it look good on the smaller devices. I changed the layout so it shows name of the columns in the each row.
